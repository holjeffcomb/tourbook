# Design Proposal: Follow vs Friend Data Model

**Status:** Superseded — folded into [`social-model.md`](./social-model.md).

This proposal evaluated adding a one-way follower/following graph alongside mutual
friendship. That idea was **rejected**: follower counts and one-way subscriptions
pull Tourbook toward a social network, away from a professional touring logbook.

The decision record and the constructive replacement (the three-axis model —
**Connections**, **Visibility**, **Publishing**) now live in a single source of truth:

➡ **[`docs/design/social-model.md`](./social-model.md)** (see Appendix A for the
follow-vs-friend decision record).
